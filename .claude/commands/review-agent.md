# 통합 리뷰

플랜 또는 코드 변경사항을 3개 도메인 에이전트로 병렬 검증합니다.

## 모드 판별

`$ARGUMENTS` 값에 따라 리뷰 모드를 결정하세요:

- **인자 없음 또는 `plan`** → 플랜 리뷰 모드
- **`code`** → 코드 리뷰 모드

## 절차

### 1. 컨텍스트 수집

**플랜 리뷰 모드:**
- 현재 대화에서 진행 중인 플랜 파일을 읽습니다
- `.claude/plans/` 디렉토리에서 가장 최근 플랜 파일을 찾아 읽습니다
- 플랜이 없으면 사용자에게 알리고 중단합니다

**코드 리뷰 모드:**
- `git diff`와 `git diff --staged`를 실행하여 변경사항을 수집합니다
- 변경이 없으면 사용자에게 알리고 중단합니다

### 2. 3개 에이전트 병렬 실행

**반드시 단일 메시지에서 3개의 Agent 도구를 동시에 호출하세요.**

각 에이전트에게 수집한 컨텍스트(플랜 내용 또는 코드 diff)를 전달하고, 아래 관점에서 리뷰를 요청합니다:

#### Agent 1: backend-cloud-engineer
```
subagent_type: backend-cloud-engineer
```
리뷰 관점:
- Lambda/DynamoDB/IoT Rule 변경 정합성
- SAM 템플릿(`template.yaml`) 호환성 — 리소스 추가/수정 시 기존 스택과 충돌 여부
- API Gateway 엔드포인트 및 CORS 설정
- 배포 영향도 — 기존 운영 중인 서비스에 미치는 영향
- config.py 가구/리전 매핑 정합성
- 기존 규칙 준수 여부 (구 API m45e239wsj 삭제 금지, Auth 스택 배포 규칙 등)

응답 형식:
```
- 영향도: [상/중/하/없음]
- 발견 사항: [구체적 이슈 또는 "이상 없음"]
- 권장 조치: [필요한 조치 또는 "없음"]
```

#### Agent 2: backend-edge-engineer
```
subagent_type: backend-edge-engineer
```
리뷰 관점:
- MQTT 토픽/페이로드 호환성 — 클라우드 변경이 엣지 서버의 MQTT 구독/발행에 영향을 주는지
- Flask 코드 연동 — 엣지 서버에서 수정이 필요한 부분이 있는지
- 엣지-클라우드 인터페이스 변경 여부 — shadow state 구조, 커맨드 포맷 등
- 엣지 서버 코드베이스(`/Users/wm-mac-01/Documents/matterhub-flask/matterhub-flask`)와의 호환성

응답 형식:
```
- 영향도: [상/중/하/없음]
- 발견 사항: [구체적 이슈 또는 "이상 없음"]
- 권장 조치: [필요한 조치 또는 "없음"]
```

#### Agent 3: frontend-integrator
```
subagent_type: frontend-integrator
```
리뷰 관점:
- API 응답 포맷 호환성 — 프론트엔드에서 사용 중인 API 응답 구조가 변경되는지
- 인증 토큰 처리 — Cognito JWT(id_token) 관련 변경 영향
- UI 변경 필요 여부 — 새로운 데이터 필드, 삭제된 필드, 구조 변경 등
- 프론트엔드 코드베이스(`/Users/wm-mac-01/Documents/admin_Solution/WhatsMatter-admin_solution`)와의 호환성

응답 형식:
```
- 영향도: [상/중/하/없음]
- 발견 사항: [구체적 이슈 또는 "이상 없음"]
- 권장 조치: [필요한 조치 또는 "없음"]
```

### 3. 통합 리포트 작성

3개 에이전트의 결과를 아래 형식으로 종합하여 사용자에게 출력하세요:

```markdown
# 통합 리뷰 리포트

## Cloud (backend-cloud-engineer)
- 영향도: [상/중/하/없음]
- 발견 사항: ...
- 권장 조치: ...

## Edge (backend-edge-engineer)
- 영향도: [상/중/하/없음]
- 발견 사항: ...
- 권장 조치: ...

## Frontend (frontend-integrator)
- 영향도: [상/중/하/없음]
- 발견 사항: ...
- 권장 조치: ...

## 종합 판정
- 전체 리스크: [상/중/하]
- 필수 조치사항: [있으면 나열, 없으면 "없음"]
- 진행 가능 여부: [진행 가능 / 조치 후 진행 / 중단 권고]
```

전체 리스크 판정 기준:
- **상**: 3개 중 하나라도 영향도 "상"이면
- **중**: 영향도 "중"이 있고 "상"은 없으면
- **하**: 모두 "하" 또는 "없음"이면
