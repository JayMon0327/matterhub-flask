# MQTT 초기 연결 안정성 강화

## 이슈 요약

Konai 납품 허브에서 재부팅 후 MQTT 연결이 TimeoutError 5회 발생 후 서비스가 crash하는 문제 해결.
부팅 직후 네트워크 미준비 상태에서 연결 시도 → 5회 실패 → 서비스 종료 → systemd restart → 반복 crash loop 발생.

## 변경 파일 목록

| 파일 | 변경 유형 |
|------|-----------|
| `mqtt.py` | 수정 |
| `tests/test_mqtt.py` | 수정 |

## 수정 내용

### 1. 네트워크 준비 대기 함수 (`_wait_for_network`)
- MQTT 연결 시도 전 DNS 서버(8.8.8.8:53) 접속으로 네트워크 가용성 확인
- 네트워크 미준비 시 10초 간격으로 무한 대기
- 부팅 직후 네트워크 인터페이스 초기화 지연 대응

### 2. 서비스 레벨 무한 재시도 (`_connect_with_service_retry`)
- `connect_mqtt()` 실패 시 서비스가 crash하지 않고 무한 재시도
- Exponential backoff: 10초 → 20초 → 40초 → ... 최대 120초
- 각 시도마다 구조화된 로그 출력 (`[MQTT][CONNECT] service_retry`)

### 3. main() 수정
- 기존: `connection = aws_client.connect_mqtt()` (예외 시 crash)
- 변경: `_wait_for_network()` → `_connect_with_service_retry(aws_client)` (예외 시 재시도)

### 4. 연결 체크 상수 추출
- `CONNECTION_CHECK_INTERVAL = 6` 상수 정의 (Phase 3에서 활용)

## 배포 여부

- 미배포 (코드 수정 완료, 테스트 통과)

## 테스트 결과

- `test_wait_for_network_succeeds_immediately` — 네트워크 즉시 가용 시 1회 시도로 성공 ✅
- `test_wait_for_network_retries_on_failure` — 2회 실패 후 3회째 성공, sleep 2회 호출 확인 ✅
- `test_connect_with_service_retry_succeeds_after_failures` — 2회 TimeoutError 후 3회째 연결 성공 ✅
- `test_connect_with_service_retry_exponential_backoff` — 백오프 간격 10→20→40초 확인 ✅
- 기존 테스트 3건 모두 통과 ✅
