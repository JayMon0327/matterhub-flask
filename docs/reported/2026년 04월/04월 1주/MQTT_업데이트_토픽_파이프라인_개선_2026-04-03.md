# MQTT 업데이트 토픽 파이프라인 개선

## 1. 이슈 요약

15103 허브 수동 업데이트 과정에서 `.env` 삭제 사고 발생 후, 클라우드 측에서 110대+ 허브의 업데이트 상태를 더 정확히 파악할 수 있도록 `mqtt_pkg/update.py`의 업데이트 파이프라인을 보강하였다.

주요 개선 4가지:
- **update_id 입력 검증**: git_update 명령에 update_id 누락 시 즉시 에러 반환 (클라우드 추적 불가 방지)
- **중복 update_id 방지**: `deque(maxlen=50)` 기반 최근 ID 추적, 동일 ID 재수신 시 `duplicate` ACK 반환
- **pre_commit 캡처**: 스크립트 실행 전 `git rev-parse HEAD` (full hash)로 사전 커밋 기록, status 파일 fallback
- **롤백 감지 선행 인프라**: `/tmp/update_{id}.rollback` 파일 존재 시 응답에 `rollback=True` 반영 (현재 `--skip-restart` 모드에서는 미생성, 향후 활용 대비)

## 2. 변경 파일 목록

## 3. 수정 내용

### mqtt_pkg/update.py

#### import 및 모듈 레벨 변수
- `from collections import deque` 추가
- `_recent_update_ids: deque = deque(maxlen=50)` — 최근 처리한 update_id 추적
- `_recent_ids_lock = threading.Lock()` — 스레드 안전 보장

#### handle_update_command() 수정
- `update_id`를 `(message.get("update_id") or "").strip()`으로 안전하게 추출
- `command in (None, "git_update")` 이고 `update_id` 없으면 `send_error_response()` 호출 후 즉시 반환
- `set_env`, `bundle_update`, `bundle_check`는 update_id 없어도 기존 동작 유지
- update_id가 `_recent_update_ids`에 이미 존재하면 `send_immediate_response(status="duplicate")` 반환

#### execute_update_async() 수정
- .env 체크 직후, 스크립트 실행 전 `git rev-parse HEAD` (full hash)로 `pre_commit` 캡처
- status 파일 읽기 후 `result.setdefault("pre_commit", pre_commit)` — status 파일 값 우선, 없을 때 Python 캡처값 사용
- `/tmp/update_{update_id}.rollback` 파일 감지 → `result["rollback"] = True`, `result["success"] = False` 설정
- 롤백 감지는 현재 `--skip-restart` 모드에서 미동작하는 선행 인프라임을 주석으로 명시

### tests/mqtt_pkg/test_update.py

| 테스트 | 검증 내용 |
|--------|-----------|
| `test_rejects_missing_update_id` | update_id 없는 git_update → send_error_response 호출, 큐 비어있음 |
| `test_rejects_duplicate_update_id` | 동일 update_id 2회 전송 → 첫 번째만 큐 진입, 두 번째는 duplicate ACK |
| `test_set_env_without_update_id_still_queued` | set_env는 update_id 없어도 큐 진입 (기존 동작 보존) |
| `test_rollback_detection_in_response` | rollback 파일 존재 시 result에 rollback=True, success=False |
