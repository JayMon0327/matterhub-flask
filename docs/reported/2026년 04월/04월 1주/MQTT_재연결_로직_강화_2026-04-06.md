# MQTT 재연결 로직 강화

## 이슈 요약

MQTT 연결 끊김 후 `check_mqtt_connection()`이 `MAX_RECONNECT_ATTEMPTS=5` 초과 시 영구적으로 재연결을 포기하는 문제 해결. 서비스는 running이지만 MQTT 미연결 상태(zombie)로 entity_changed 발행이 중단됨.

## 변경 파일 목록

| 파일 | 변경 유형 |
|------|-----------|
| `mqtt_pkg/runtime.py` | 수정 |
| `tests/test_runtime_reconnect.py` | 신규 |

## 수정 내용

### 1. 재연결 무한 재시도 + 점진적 백오프
- 기존: `MAX_RECONNECT_ATTEMPTS=5` 초과 시 `return False` (영구 포기)
- 변경: 포기하지 않고 계속 재시도. 5회 초과 시 점진적 백오프 적용
  - `RECONNECT_BACKOFF_THRESHOLD = 5` (즉시 재시도 횟수)
  - `RECONNECT_BASE_DELAY = 10` (백오프 시작 대기)
  - `RECONNECT_MAX_DELAY = 300` (최대 5분)
  - 백오프 계산: `min(10 * 2^(n-1), 300)` 초

### 2. `_pending_resubscribe` 플래그
- `on_connection_resumed` 콜백에서 `session_present=False`면 플래그 설정
- `check_mqtt_connection()` 호출 시 플래그 확인 → resubscribe 수행
- SDK 자동 재연결 후 세션 유실 시 토픽 구독 복구 보장

### 3. `on_interrupted` 로그 개선
- 기존: `reconnect_attempt=N/5` (고정 한도 표시)
- 변경: `reconnect_attempt=N` (한도 없이 현재 시도 횟수만 표시)
