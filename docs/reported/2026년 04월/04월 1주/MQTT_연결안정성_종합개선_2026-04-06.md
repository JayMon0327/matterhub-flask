# MQTT 연결 안정성 종합 개선

## 이슈 요약

Konai 납품 MatterHub 허브에서 MQTT 연결 불안정 이슈가 발생.
재부팅 후 entity_changed 미발행, 간헐적 연결 후 수 분 내 끊김, delta 명령 무응답.
4가지 근본 원인을 분석하고 Phase 1~5로 단계적으로 해결.

### 근본 원인

| # | 원인 | 코드 위치 | 영향 |
|---|------|-----------|------|
| 1 | 초기 연결 실패 시 서비스 crash | `mqtt.py:128` | 부팅 후 crash loop |
| 2 | 재연결 5회 후 영구 포기 | `runtime.py:229-231` | zombie 서비스 |
| 3 | 끊김 감지 지연 (5분/60초) | `runtime.py:98`, `mqtt.py:150` | 무음 실패 |
| 4 | 발행 실패 시 silent return | `publisher.py:19-21`, `state.py:130-131` | 진단 불가 |

### MQTT 원격 업데이트 불가 사유
- `SUBSCRIBE_MATTERHUB_TOPICS` 기본값 `"0"` → update 토픽 미구독
- MQTT 연결 자체가 불안정 → 명령 수신 불가
- SSH 기반 원나잇 패치 스크립트로 전달

## 수정 내용

### Phase 1: 초기 연결 안정성 강화
- `_wait_for_network()`: 부팅 후 네트워크 준비 대기 (DNS 8.8.8.8:53 접속 확인)
- `_connect_with_service_retry()`: 서비스 레벨 무한 재시도 (exponential backoff 10~120초)
- `main()`에서 crash 없이 연결될 때까지 대기

### Phase 2: 재연결 로직 강화
- `MAX_RECONNECT_ATTEMPTS=5` 제한 제거 → 무한 재시도
- 5회 초과 시 점진적 백오프 (10~300초)
- `_pending_resubscribe` 플래그: SDK 자동 재연결 후 토픽 재구독 보장

### Phase 3: 연결 상태 감지 개선
- `keep_alive_secs`: 300초(5분) → 30초 (AWS IoT Core 최소값)
- connection check 주기: 60초 → 30초
- `is_connected()` False 감지 시 즉시 재연결 트리거

### Phase 4: 발행 안정성 개선
- `publisher.py`: `is_connected()` 이중 체크 + 구조화된 SKIP 로그
- `state.py`: `_log_disconnected_once()` — 30초 빈도 제한 끊김 로그

### Phase 5: 코나이 전달 산출물
- `patch_mqtt_stability.sh`: SSH 원나잇 패치 (백업/롤백/dry-run 지원)
- 문제 원인 분석 문서 + 패치 적용 가이드

## 배포 여부
