# MQTT 연결 상태 감지 개선

## 이슈 요약

MQTT 연결 끊김 감지가 느려(최대 5분) 무음 실패 기간이 길었던 문제 해결.
keep_alive 5분 → 30초, connection check 60초 → 30초로 단축하고 끊김 즉시 감지 로직 추가.

## 변경 파일 목록

| 파일 | 변경 유형 |
|------|-----------|
| `mqtt_pkg/runtime.py` | 수정 |
| `mqtt.py` | 수정 |

## 수정 내용

### 1. keep_alive_secs 단축
- 기존: `keep_alive_secs=300` (5분)
- 변경: `keep_alive_secs=30` (30초, AWS IoT Core 최소값)
- 효과: 서버 측 끊김 감지 5분 → ~45초(30초 + 1.5x timeout)

### 2. connection check 주기 단축
- 기존: `connection_check_counter >= 12` (5초 × 12 = 60초)
- 변경: `CONNECTION_CHECK_INTERVAL = 6` (5초 × 6 = 30초)

### 3. 즉시 감지 로직
- main loop에서 `is_connected() == False` 감지 시 `connection_check_counter`를 즉시 threshold로 설정
- 다음 루프(5초 후)에서 바로 `check_mqtt_connection()` 호출
