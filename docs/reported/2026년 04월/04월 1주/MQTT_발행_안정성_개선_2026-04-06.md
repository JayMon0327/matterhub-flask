# MQTT 발행 안정성 개선

## 이슈 요약

MQTT 연결 끊김 시 entity_changed 발행이 silent return으로 실패하여 장애 진단이 불가능했던 문제 해결.
`publisher.py`와 `state.py`에 구조화된 로그를 추가하고, 연결 상태 체크를 강화함.

## 변경 파일 목록

| 파일 | 변경 유형 |
|------|-----------|
| `mqtt_pkg/publisher.py` | 수정 |
| `mqtt_pkg/state.py` | 수정 |
| `tests/mqtt_pkg/test_publisher.py` | 수정 |

## 수정 내용

### 1. publisher.py 연결 상태 체크 강화
- 기존: `connection is None` 만 체크 → 연결 객체는 있지만 끊긴 경우 감지 불가
- 변경: `connection is None or not runtime.is_connected()` 로 이중 체크
- 로그: `[MQTT][PUBLISH][SKIP] topic=... type=... reason=disconnected|no_connection`

### 2. state.py 연결 끊김 로그 (30초 빈도 제한)
- `_log_disconnected_once()` 함수 추가: 30초에 1번만 로그 출력
- `publish_bootstrap_all_states()` + `publish_device_state()`에 적용
- 기존 silent return → `[MQTT][BOOTSTRAP][SKIP] reason=disconnected`, `[MQTT][ENTITY_CHANGED][SKIP] reason=disconnected`

### 3. 기존 테스트 수정
- `test_publisher.py`: `is_connected()` mock 추가 (코드 변경에 따른 테스트 동기화)
